## b. 本次變動項目

| 序號 | 項目 | 改變內容 | 如何驗收 | 預期效果 |
|---|---|---|---|---|
| 1 | Pi repair 腳本會自動拉最新程式碼 | `scripts/repair-pi.sh` 現在會從 `security-final` 分支 clone 最新 code 並重新執行 `install.sh` | 在 Pi 執行 `repair-pi.sh` 後，`app/routes/auth.py` 不再關閉 SQLite 連線 | 修復後 web service 可正常啟動 |
| 2 | Cloud 預設 broker 與新增店點連動 | 「預設 Broker」頁面儲存的 broker / port / 帳號 / 密碼 / TLS 設定，會自動帶入「新增店點」表單 | 進入「店點管理 → 預設 Broker」填寫 114.55.1.51:1883 並儲存；切到「店點列表 → 新增店點」，欄位已自動帶入 | 減少重複輸入 |
| 3 | 新增店點移除裝置 ID 與角色 | 表單與列表都不再顯示 deviceId / role | 新增/編輯店點時看不到這兩個欄位 | 介面更簡潔 |
| 4 | TLS 預設不勾選 | 新增/編輯店點的「使用 TLS」與「驗證憑證」預設為未勾選；後端與連線測試也同步改為預設 false | 新增店點時兩個 TLS 選項皆未勾選 | 直接以 broker/port/帳號/密碼連入 |
| 5 | 音樂庫改為 NAS WebDAV 清單 | `LibraryClient.js` 改為呼叫 `/api/library?source=webdav`；Pi 新增 `webdav_list_music` MQTT 指令 | 進入「店點管理 → 音樂庫」，點「載入 NAS 音樂清單」，會列出 NAS 上的 MP3 | 統一從 NAS 控管音樂 |
| 6 | 保留全店同步/下載 | 音樂庫頁面保留店點選取（含全選）與「同步到選取店點」、「下載到選取店點」按鈕 | 勾選多家店點後按同步 | 批次下發 `sync` 指令 |
| 7 | OTA 支援全選 | `OtaClient.js` 改為多選店點（含全選），可批次 OTA 更新 / Rollback | 進入「店點管理 → OTA」，勾選全部店點後更新 | 多店可一鍵更新 |
| 8 | OTA 更新紀錄移到監控紀錄 | 新增「監控紀錄 → OTA 紀錄」分頁；原「店點管理 → OTA」不再顯示紀錄 | 進入「監控紀錄」可看到 OTA 紀錄分頁 | 紀錄集中管理 |
| 9 | 告警列表排版修正 | `AlertsClient.js` 改為四欄：店點 / 告警內容 / 時間 / 操作，減少寬度與留白 | 進入「監控紀錄 → 告警中心」 | 列表更緊湊 |
| 10 | Linear token 已儲存 | 使用你提供的 token 完成 `linearis auth login`，儲存在 `~/.linearis/token` | 後續可直接執行 Linear 更新 | 不再每次要 token |
| 11 | Cloud Production 重新部署 | `security-final` 已 push 並執行 `vercel --prod` | 開啟 `https://cloud-vercel-xi.vercel.app` 可看到新版 UI | 線上環境同步 |
