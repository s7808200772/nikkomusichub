## b. 本次變動項目

### 1. 修復 Cloud 無法 MQTT 偵測 Pi 的問題
- **根因**：Pi 的 `nikko.env` 中 `NIKKO_MQTT_COMMAND_SECRET` 與 `NIKKO_MQTT_TOPIC_PREFIX` 是隨機值（例如 `nikko-78595cafc1d82d76e48e6e6f`），與 Cloud 使用的 `topup30%off` / `nikko` 不一致，導致簽名驗證失敗、topic 對不上，Pi 收不到/回不了指令。
- **修正**：
  - `install.sh` 現在會**強制覆寫**現有 `nikko.env` 的 `COMMAND_SECRET`、`TOPIC_PREFIX`、`TLS`、`TLS_VERIFY` 為統一預設值（`topup30%off` / `nikko` / `0` / `0`）。
  - `app/config.py` 的 fallback 預設值也改為 `114.55.1.51:1883`、TLS off、`admin` / `topup30%off`。
- **驗收**：在 Pi 重新執行 `sudo bash scripts/repair-pi.sh` 後，Cloud 店點管理的「測試 MQTT 連線」應顯示成功。

### 2. Pi UI / 功能調整
- **首頁儀表右側顯示 Store ID**：在「店家資訊」卡片新增 Store ID 欄位。
- **網路設定不再跳出一堆訊息**：原本測試/儲存時把整段 `nmcli` 輸出塞進 4 秒 toast，導致訊息一閃而過。現在 toast 只顯示摘要，完整輸出顯示在表單下方的持續訊息框，方便截圖與查看。
- **系統設定卡片佈局調整**：縮短「同步設定」與「WebDAV / 手動同步」卡片，把「系統初始化」與「網路設定」緊接上方卡片排列，減少空白。
- **同步結果說人話**：
  - Dry-run：顯示「將從 NAS 同步 N 個檔案到本地…」並說明方向與本地刪除檔案會被重新下載。
  - 立即同步：顯示總曲目數、新增、更新、刪除、耗時、平均速度等資訊。
- **版本更新頁面有內容了**：修正 Jinja2 語法錯誤（`entry.items` 被當成 dict 方法），現在會正確列出 changelog。

### 3. Cloud UI / 功能調整
- **OTA 與音樂庫的店點列表會刷新**：進入頁面或切回頁面時會重新從伺服器取得店點列表；新增/編輯/刪除店點後也會即時同步到其他分頁。
- **音樂庫 NAS 載入有提示且會保存**：
  - 載入中、成功、失敗都會顯示明確訊息，不再只是轉圈圈。
  - 載入成功的清單會保存，再次進入頁面會先顯示上次清單與「最後更新時間」。
- **總覽控制台選取/全選欄位加寬**：避免「選取」「全選」兩個字換行。
- **OTA 非同步 job 狀態持久化**：job 狀態改存到 Supabase（未設定時回退記憶體），避免 Vercel 實例切換導致狀態遺失。
- **GitHub Token 改走伺服器端**：新增 `/api/github-version`，OtaClient 改呼叫此 route，token 不會暴露給瀏覽器；已將 `.env.local` 的 `NIKKO_GITHUB_TOKEN` 同步到 Vercel Production / Preview。

## c. 目前遺留問題

### ⭐⭐⭐⭐⭐ 關鍵（依使用者決定暫不處理）

1. **硬編碼預設憑證**
   - 影響：repo 內可看到 `topup30%off` 等預設密碼
   - 風險：設備與 Cloud 易被接管
   - 狀態：使用者決定維持現狀，不強制修改

2. **Pi Web UI 維持 `0.0.0.0:8080` 與預設密碼**
   - 影響：任何可連上 Pi 網路者都能進入管理介面
   - 風險：未授權控制
   - 狀態：使用者決定不改

3. **MQTT 維持 plaintext 且正式使用**
   - 影響：憑證與指令在網路上明文傳送
   - 風險：中間人可竊聽/重送指令
   - 狀態：使用者決定維持 plaintext

### ⭐⭐⭐⭐ 高風險（仍待處理）

4. **Cloud 管理員密碼明文比對、無 hash、rate limit 可被繞過**
   - 影響：暴力嘗試與外洩風險
   - 風險：Cloud 控制台被攻破

5. **缺少全域 middleware / CSRF 防護**
   - 影響：每條 route 各自驗證，容易遺漏
   - 風險：跨站偽造請求

6. **Pi 備份路徑 traversal**
   - 影響：認證使用者可下載 `BASE_DIR` 下任意檔案
   - 風險：DB、設定檔外洩

7. **Pi 音樂庫檔名 XSS**
   - 影響：惡意 MP3 檔名可執行 JS
   - 風險：竊取管理員 session

8. **OTA / repair 腳本無簽章驗證**
   - 影響：`git pull` + `pip install` 無 hash/分支鎖定
   - 風險：GitHub 帳號被盜可植入惡意程式

### ⭐⭐⭐ 中等風險

9. **OTA 雖非同步但仍可能超時**：更新時間長時建議再觀察。
10. **GitHub token 已設定但仍需確認 `/api/github-version` 運作**。
11. **Supabase jobs 表遷移檔已新增，需確認部署環境已執行 migration**。
