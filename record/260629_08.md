## b. 本次變動項目

| 序號 | 項目 | 改變內容 | 如何驗收 | 預期效果 |
|---|---|---|---|---|
| 1 | Pi 音樂庫按鈕清理 | `app/templates/dashboard.html`、`app/static/style.css` 簡化音樂庫區塊按鈕，移除重複或低使用率入口 | 開啟 Pi `/dashboard`，音樂庫相關按鈕精簡且可正常點擊 | 降低操作困惑 |

| 4 | Cloud OTA 版本號顯示修正 | `cloud-vercel/app/ota/OtaClient.js` 修正版本解析與顯示邏輯，避免 `undefined` 或錯誤版本號 | 開啟 `/ota`，每家店的目前版本與可更新版本正確顯示 | 管理員可正確判斷更新需求 |
| 5 | Cloud Dashboard human-readable 輸出 | `cloud-vercel/app/DashboardClient.js` 將店點狀態、MQTT 響應改以易讀格式呈現 | 開啟 `/`，卡片與列表資訊一目瞭然 | 減少誤讀 |
| 6 | Cloud API 按鈕與響應格式化 | 新增 `cloud-vercel/components/ApiButton.js`、`ResponseFormatter.js`、`lib/fetchUtils.js`，統一按鈕載入狀態與錯誤/成功訊息格式化 | 在 `/commands`、`/stores` 執行指令或測試連線時，按鈕有載入回饋且響應易讀 | 降低誤操作與排錯時間 |
| 7 | Cloud 各頁面 UI/UX  polish | `cloud-vercel/app/commands/CommandsClient.js`、`stores/StoresClient.js`、`globals.css` 等調整表格、間距、按鈕層級 | Cloud 主要頁面操作順暢、視覺一致 | 提升管理效率 |
| 8 | 推送 `security-final` 並部署生產環境 | `git push origin security-final` + `vercel --prod --yes` 成功 | 開啟生產網址可看到最新版本 | 線上環境同步 |
