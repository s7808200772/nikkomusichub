## c. 目前遺留問題

以下為綜合審查後的優先清單。星數 = 重要性 + 風險 + 優先度（1–5 星）。

### ⭐⭐⭐⭐⭐ 關鍵（建議立即處理）

1. **硬編碼預設憑證**（Cloud/Pi/install.sh）
   - 影響：`topup30%off`/`admin` 寫在原始碼與安裝腳本中，任何人拿到 repo 就知道帳密
   - 風險：設備與 Cloud 被輕易接管

2. **Pi Web UI 監聽 `0.0.0.0:8080` 且使用預設密碼**
   - 影響：只要連得上 Pi（LAN/Tailscale）就能進入管理介面
   - 風險：未授權遠端控制、服務被關閉或資料被竊取

3. **Cloud 管理員密碼以明文比對、無 hash、rate limit 易被繞過**
   - 影響：密碼外洩即全權限洩漏；短時間內可被暴力嘗試
   - 風險：Cloud 控制台被攻破，可對所有店點下指令

4. **危險指令一鍵執行、無二次確認**
   - 影響：重開機、OTA、Rollback、Sync 等按鈕點了就發送
   - 風險：誤觸或被惡意利用導致店點停機

5. **MQTT 預設 plaintext、TLS 驗證關閉**
   - 影響：broker 憑證與指令在網路上傳送明文
   - 風險：中間人可竊聽、重送指令

6. **Pi 備份下載存在路徑 traversal**
   - 影響：認證使用者可下載 `BASE_DIR` 下任意檔案（如 DB、設定檔）
   - 風險：敏感資料外洩

7. **Pi 音樂庫檔名可觸發 XSS**
   - 影響：惡意命名的 MP3 檔案可在管理員點擊時執行 JavaScript
   - 風險：竊取管理員 session 或執行未授權操作

### ⭐⭐⭐⭐ 高風險

8. **缺少全域認證 middleware / CSRF 防護**（Cloud）
   - 影響：每條 route 各自驗證，容易遺漏；跨站偽造請求風險較高

9. **`/api/command` 單一指令未驗證 `commandKey`**（Cloud）
   - 影響：擁有 session 即可發送任意命令鍵
   - 風險：雖然 Pi 端有白名單，但仍增加攻擊面

10. **Pi 需要 passwordless sudo 但安裝腳本未設定**
    - 影響：`systemctl` 操作會無聲失敗，導致服務控制、同步、watchdog 不可靠

11. **WiFi 密碼明文存於 SQLite 且未實際套用連線**
    - 影響：密碼可從 DB 讀取；填寫的 WiFi 設定沒有效果

12. **OTA / repair 腳本下載遠端程式碼無簽章驗證**
    - 影響：`git pull` + `pip install` 無 hash、無分支鎖定
    - 風險：GitHub 帳號被盜或中間人可植入惡意程式

### ⭐⭐⭐ 中等風險

13. **文件與 `.env.example` 的 MQTT/TLS/帳密預設值不一致**
    - 影響：新開發者或維運人員依文件設定會連到錯誤 broker 或啟用 TLS

14. **Cloud Settings API 回傳明文預設密碼**
    - 影響：瀏覽器可看到預設 MQTT / NAS 密碼

15. **`localStorage` 儲存店點與設定密碼**（Cloud localStorage fallback）
    - 影響：未設定 Supabase 時，密碼存在瀏覽器 plaintext

16. **Alert 欄位 `storeId` / `store_id` 可能不一致**
    - 影響：告警無法正確對應到店點名稱或寫入失敗

17. **OTA timeout 僅 20 秒，實際更新可能更久**
    - 影響：Cloud 顯示失敗但 Pi 仍在更新，狀態不同步

18. **批次指令無併發控制**
    - 影響：店點多時同時開大量 MQTT 連線，可能觸發連線上限或 function timeout

### ⭐⭐ 低風險 / 維護性

19. `/api/version` 未認證（資訊洩漏）
20. GitHub API 未使用 token，可能觸發 rate limit
21. Cookie 未使用 `__Host-` 前綴
22. `ensureStorePrefix("store")` 會產生無效 `store-` ID
23. Pi network priority 失敗未回報使用者
24. `change_password.html` 的 `minlength` 與後端要求不一致
25. `install.sh` 未複製/啟用 backup service/timer（與 README 矛盾）
