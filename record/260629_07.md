## b. 本次變動項目

### 1. Pi Network Watchdog（本次主要功能）
- **Watchdog 腳本**：`app/systemd/nikko-network-watchdog.sh`
  - 每分鐘檢查 default gateway、外網 IP（1.1.1.1 / 8.8.8.8）、DNS（google.com / login.tailscale.com）
  - 連續失敗 3 次：重啟 NetworkManager（fallback dhcpcd / networking）
  - 連續失敗 5 次：重啟 tailscaled
  - 連續失敗 8 次：自動 reboot，但 30 分鐘內只 reboot 一次（cooldown）
  - 所有動作寫入 `logger -t nikko-network-watchdog`，可用 `journalctl -t nikko-network-watchdog` 查看
- **Systemd 單元**：`nikko-network-watchdog.service` + `nikko-network-watchdog.timer`（每分鐘執行）
- **Pi command handlers**：`network_watchdog_install`、`network_watchdog_disable`、`network_watchdog_status`、`network_watchdog_logs`
- **Cloud API**：`/api/watchdog` 支援單台與批量 (`storeIds`) 操作
- **Cloud UI**：在「店點管理」頁面新增每列四顆按鈕（安裝/更新、狀態、Log、停用），以及表格上方「批量安裝/停用看門狗」按鈕；狀態與 Log 以彈窗顯示

**驗收方式**：
- 登入 Cloud → 店點管理 → 選一家 Pi → 點「安裝/更新看門狗」→ 應顯示成功
- 點「看門狗狀態」應看到 timer_active、timer_enabled、service_exists、fail_count、last_action
- 點「看門狗 Log」應看到最近 50 行 journalctl
- 在 Pi 執行 `systemctl list-timers nikko-network-watchdog.timer` 應看到每分鐘執行

### 2. Pi 網路優先順序權限修復
- `install.sh` 的 sudoers 新增：
  - `nmcli connection modify/up`、`nmcli device set`
  - systemctl start/stop/restart/enable/disable for `NetworkManager`、`dhcpcd`、`networking`、`tailscaled`、`nikko-network-watchdog`
- `app/routes/network.py` 的 `nmcli connection modify` 改以 `sudo` 執行
- 設定卡片間距再縮小，MQTT 與店家資訊、WebDAV 與網路設定更緊密

### 3. 音樂庫刪除真正刪檔
- 刪除按鈕現在會從 `MUSIC_DIR` 移除實體 MP3 檔案
- 若正在播放該檔案，會先停止再刪除，並重新載入播放清單
- UI 會即時反映刪除結果

### 4. 測試同步與立即同步改善
- **測試同步**：dry-run 前用 `rclone lsf` 比對遠端與本地 MP3 清單，正確顯示「將從 NAS 下載 N 個檔案（包含你剛刪除的本地檔案）」
- **立即同步**：改用 `--progress --stats-one-line`，UI 每秒輪詢 `/api/webdav/sync-progress`，即時顯示百分比、目前檔案、速度、ETA，不再卡住 0%

### 5. 版本更新與首頁問題修復
