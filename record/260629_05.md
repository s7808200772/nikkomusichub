## b. 本次變動項目

### 1. Cloud UI / 功能修正
- **遠端 log 載入按鈕對齊**：按鈕與店點選擇框現在同一水平線上。
- **總覽控制台重排**：中央控制台已移至最底部，位於店點指令控制台之下。
- **輸出面板顯示結果**：點選任何店點指令後，右側輸出區會顯示實際回傳結果或錯誤訊息，不再只有 `{}`。
- **NAS 音樂清單載入與持久化**：修正 MQTT 命令 payload 導致的 `Connection refused: Not authorized` 錯誤；載入後的清單會保留在頁面上並顯示最後更新時間，不用每次重新整理。
- **危險指令二次確認**：重開機、OTA、Rollback、Sync、重啟播放服務等按鈕會先跳出確認對話框才執行。
- **localStorage 不再儲存密碼**：Supabase 未設定時，瀏覽器本地只保存非敏感資料，MQTT / WebDAV 密碼不會被記錄。
- **`store-` 前綴驗證**：輸入 `store` 等會產生空後綴的情況會被擋下並提示。

### 2. Cloud 後端 / 安全強化
- **`/api/command` 驗證 commandKey**：只允許白名單內的指令鍵。
- **OTA 非同步作業**：改為建立 job 並輪詢狀態，timeout 延長為 120 秒，避免更新還沒完成就回報失敗。
- **批次指令併發控制**：同時最多 5 個店點連線，避免連線數爆炸。
- **`__Host-` cookie**：Cookie 名稱改為 `__Host-nikko_cloud_token`，並設定 `secure` + `path: /`。
- **GitHub token 支援**：OTA 版本檢查會優先使用 `NIKKO_GITHUB_TOKEN`，減少 rate limit 風險。
- **MQTT 連線偵測加強**：改發輕量 `status_system`、timeout 30 秒、錯誤訊息更具體（區分 timeout / broker / auth / topic 不符）。

### 3. Pi UI 調整
- **右側卡片順序**：改為 rclone → rclone 版本 → mpv → mpv 版本 → 音樂服務。
- **播放器控制台標籤**：音量標籤改為「播放音量」並置於滑桿下方；進度條下方新增「播放進度」。
- **靜音按鈕**：靜音時音量滑桿歸 0，取消靜音時恢復原本音量，圖示與 title 即時更新。
- **系統設定卡片重排**：「系統初始化」獨立成卡但仍在同步設定下方；「網路設定」移至 WebDAV 手動同步卡片下方，減少空白。
- **Remote Music Path**：移除強制前導 `/`，讓使用者在輸入框直接填寫完整路徑（如 `/NikkoMusic`）。
- **WebDAV 按鈕**：「儲存設定 / 測試同步 / 立即同步」三顆按鈕現在同一列、等寬、無分隔線。
- **Dry-run 摘要**：測試完成後會顯示「將同步 X 個檔案，無錯誤」或「發現錯誤」等清楚摘要。

### 4. Pi 後端 / 安裝 / 文件
- **版本更新頁面**：補上近期 changelog（網路測試、WebDAV、播放器、MQTT、儀表板、install.sh 調整等）。
- **網路設定先測試再套用**：新增「測試連線」按鈕，驗證 `nmcli`、連線類型、WiFi SSID；通過後才啟用「儲存並套用」按鈕；失敗會明確回傳錯誤。
- **修改密碼前端驗證**：`minlength` 改為 12，並加上與後端一致的字元要求提示。
- **install.sh 免密 sudo**：自動建立 `/etc/sudoers.d/nikko-music`，允許服務帳號執行必要的 systemd 指令。
- **install.sh 備份排程**：複製並啟用 `nikko-music-backup.service` / `.timer`。
- **`.env.example` 一致性**：預設值改為與 `install.sh` 相同（broker `114.55.1.51:1883`、TLS off、帳密 / command secret / topic prefix 統一）。

## c. 目前遺留問題

### ⭐⭐⭐⭐⭐ 關鍵（依使用者決定暫不處理）

1. **硬編碼預設憑證保留在程式碼中**
   - 影響：任何人拿到 repo 就知道預設帳密與 MQTT command secret
   - 風險：設備/Cloud 易被接管
   - 狀態：使用者決定維持現狀，登入後不強制修改密碼

2. **Pi Web UI 維持監聽 `0.0.0.0:8080` 且使用預設密碼**
   - 影響：可連上 Pi 網路者即可進入管理介面
   - 風險：未授權控制、服務被關閉
   - 狀態：使用者決定不改

3. **MQTT 維持 plaintext 且正式使用**
   - 影響：憑證與指令在網路上明文傳送
   - 風險：中間人可竊聽/重送指令
   - 狀態：使用者決定維持 plaintext

### ⭐⭐⭐⭐ 高風險（仍待處理）

4. **Cloud 管理員密碼仍以明文比對、無 hash、rate limit 可被繞過**
   - 影響：暴力嘗試門檻低；密碼外洩即全權限
   - 風險：Cloud 控制台被攻破

5. **缺少全域 middleware / CSRF 防護**
   - 影響：每條 route 各自驗證，容易遺漏
   - 風險：跨站偽造請求與未授權操作

6. **Pi 備份下載路徑 traversal**
   - 影響：認證使用者可下載 `BASE_DIR` 下任意檔案
   - 風險：DB、設定檔外洩

7. **Pi 音樂庫檔名 XSS**
   - 影響：惡意 MP3 檔名可在管理員點擊時執行 JS
   - 風險：竊取 session 或執行未授權操作

8. **OTA / repair 腳本無簽章驗證**
   - 影響：`git pull` + `pip install` 無 hash/分支鎖定
   - 風險：GitHub 帳號被盜可植入惡意程式

### ⭐⭐⭐ 中等風險

9. **文件與 `README`、`AGENTS.md` 仍有過時資訊**
   - 影響：新開發者/維運人員依舊文件操作會出錯

10. **OTA timeout 雖延長但仍同步阻塞**
    - 影響：更新時間長時仍可能超時

11. **批次指令併發雖限制但 job 狀態仍存記憶體**
    - 影響：Vercel serverless 實例切換時 job 可能遺失

12. **GitHub API token 需手動設定**
    - 影響：未設定時仍可能觸發 rate limit

### ⭐⭐ 低風險

13. Cookie 未使用 `__Host-` 以外的額外保護（已完成 __Host-）
14. `ensureStorePrefix` 空後綴（已修正）
15. Pi network priority 失敗未回報（已回報）
16. `change_password.html` 驗證不一致（已修正）
17. `install.sh` 未啟用 backup（已啟用）
