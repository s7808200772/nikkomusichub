## b. 本次變動項目（寫給使用者）

1. **店點管理表格欄位置中**
   - 已登錄店點的「選取 / 店點 / Broker / 使用者 / 連線 / 看門狗 / 操作」全部改為水平置中，視覺更整齊。
   - 驗收：進入 `/stores`，表格內容與表頭都在各自欄位中央。

2. **店點指令控制台「播放 / 繼續」真正有效**
   - 之前點藍色播放 icon 會顯示成功，但如果 Pi 上的 mpv 已經在跑且暫停，並不會繼續播放。
   - 現在 Cloud 發出 `player_play` 時，Pi 會判斷：已暫停則繼續播放、尚未啟動則啟動、已在播放則回報正常運作。
   - 驗收：選取店點 → 點藍色播放 icon → Pi 實際開始或繼續播放音樂。

3. **儀表板 / 系統資訊卡片不再遮字**
   - 回傳內容（IP、主機名、同步訊息等）過長時，原排版會被截斷或遮住。
   - 改為單欄上下排列、自動換行，並給卡片最小寬度，確保所有文字都看得見。
   - 驗收：在 `/commands` 點「儀表板」或「系統資訊」，展開後所有欄位完整可讀。

4. **最近批量任務穩定顯示最多三條**
   - 之前批量操作後紀錄可能因輪詢或實例切換而閃爍消失。
   - 現在瀏覽器會同時保留最近三筆任務，並與伺服器資料合併；即使短暫連線異常也不會清空。
   - 驗收：執行一次批次指令 → 中央控制台「最近批量任務」持續顯示該紀錄，滿三筆後最新一筆覆蓋最舊一筆。

5. **Pi 操作狀態卡片間距對齊**
   - 「操作狀態」卡片與下方「店家資訊 / 網路設定 / WebDAV 手動同步」三張卡片的垂直距離，現在與其他卡片間距一致。
   - 驗收：進入 Pi `/settings`，操作狀態與第一排卡片間距均勻。

6. **Pi 首頁藍色音樂卡片寬度對齊**
   - 藍色「目前播放」卡片原本因舊樣式殘留的 `align-items: center` 導致寬度沒有撐滿，看起來比控制列與音量列窄。
   - 修正後藍色卡片與控制列、音量列同寬。
   - 驗收：Pi 首頁三個元素（藍色卡片、控制按鈕列、音量列）左右對齊。

7. **Pi 系統初始化按鈕確實靠左**
   - 之前 `button.icon-left` 的全域樣式把按鈕內容置中，覆蓋了初始化區域的靠左設定。
   - 新增專屬規則讓系統初始化六顆按鈕的圖示與文字都靠左對齊。
   - 驗收：進入 Pi `/settings`，系統初始化六顆按鈕內容靠左。
