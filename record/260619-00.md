## f. 序號 2 執行完後的驗收方式

本輪對應「序號 2」的整體目標是讓系統穩定可商用。驗收時你應該看到：

1. 登入 Cloud Dashboard，總店數、線上、異常、未確認告警數字正確。
2. `/commands` 可勾選多家店點發送「同步」，並看到成功/失敗/無回應計數。
3. 關閉某家 Pi 的 MQTT agent 超過 90 秒，Cloud `/alerts` 出現 offline 告警。
4. `/library` 可載入各店音樂檔案，勾選店點按「同步選取店點」後，Pi 開始同步。
5. `/ota` 可對單店發送 OTA 更新；更新後 Pi `/api/version` 的 git commit 變新。
6. Pi Dashboard 的 QNAP/Tailscale 燈號在網路正常時顯示綠色。
7. 故意斷開 QNAP WebDAV 執行同步，本地音樂不會被刪除，播放不中斷。
8. 執行 `python scripts/test-suite.py`（Pi）與 `node cloud-vercel/scripts/test-suite.js`（Cloud）能看到逐項 PASS/FAIL。
