你現在進入「可落地產品開發模式」。請協助我設計並實作一套 Raspberry Pi 門市音樂管理系統，系統名稱暫定：

NikkoMusicHub｜日光音樂節點

一、產品目標

我要把 Raspberry Pi 變成門市用的「雲端更新、本地播放、可遠端管理」音樂節點。

未來可能會有 30～100 間店，不可能每台都用 SSH 手動控制。因此我要做一個 Web 管理介面，之後我只需要先完成最基本的人工設定，剩下全部透過瀏覽器按按鈕完成。

二、使用前提

我人工只會先做以下事情：

把 Raspberry Pi OS 安裝好。
設好 Wi-Fi 或有線網路。
打開 SSH。
裝好並登入 Tailscale。
確認我可以透過 Tailscale IP 連到這台 Raspberry Pi。

從這一步之後，我希望所有操作都透過瀏覽器完成。

三、核心使用方式

我希望未來可以在瀏覽器輸入：

http://TAILSCALE_IP

例如：

http://100.x.x.x:8080

然後進入 NikkoMusicHub 的 Web 管理介面。

第一次進入時，輸入：

預設管理帳號。
預設管理密碼。
Dropbox token。
Dropbox 音樂資料夾路徑，例如 dropbox。
本地音樂資料夾路徑，預設 /srv/nikko-music/music。

之後所有操作都在網頁上按鈕完成。

四、系統建議架構

請優先使用適合 Raspberry Pi 的輕量架構：

後端：Python FastAPI 或 Flask。
前端：簡單 HTML + CSS + JS 即可，不需要複雜框架。
本地資料儲存：SQLite 或 JSON config。
系統服務管理：systemd。
雲端同步：rclone。
音樂播放：mpv。
mpv 控制：使用 mpv IPC socket，例如 /tmp/nikko-mpv.sock。
遠端存取：透過 Tailscale IP，不做公網開放。

請不要做過度複雜架構。此系統要穩定、可維護、好部署。

B. 初始化安裝頁

每個功能做成按鈕，不要要求我 SSH 下指令。

需要按鈕：

update + upgrade
執行 sudo apt update && sudo apt upgrade -y。
安裝 rclone
自動安裝 rclone，完成後顯示版本。
安裝 mpv
自動安裝 mpv，完成後顯示版本。
建立音樂資料夾
建立：
/srv/nikko-music/music
/srv/nikko-music/logs
/srv/nikko-music/scripts
設定 Dropbox
允許我在 Web 表單貼上 Dropbox rclone token JSON，並輸入 Dropbox remote name、雲端路徑、本地路徑。
需要安全儲存 token，不要明文顯示在頁面上。
rclone config 權限必須限制，例如 chmod 600。
測試 Dropbox 連線
執行類似 rclone lsf dropbox，顯示成功或錯誤。
同步音樂
執行 rclone sync，把 Dropbox 指定資料夾同步到 /srv/nikko-music/music。
安裝 / 更新播放服務
建立 systemd service，讓 mpv 開機自動播放。
啟用開機自動播放
enable music player service。
停用開機自動播放
disable music player service。

每個按鈕都要：

顯示執行中。
顯示成功 / 失敗。
顯示完整 log。
避免重複點擊造成衝突。
指令要 idempotent，可重複執行不破壞系統。

C. Dropbox 設定頁

欄位：

remote name，預設 dropbox。
Dropbox token JSON。
Dropbox 音樂路徑，預設 dropbox。
本地音樂路徑，預設 /srv/nikko-music/music。
同步模式：sync。
檔案格式限制：只同步 .mp3 / .MP3。
同步排程：
每天固定時間，例如 03:00。
開機後延遲同步，例如開機 2 分鐘後同步一次。
手動同步按鈕。
dry-run 測試按鈕。
最近同步紀錄。

同步邏輯：

Dropbox 是母資料庫。
Pi 本地是副本。
Dropbox 新增，Pi 下載。
Dropbox 修改，Pi 覆蓋。
Dropbox 刪除，Pi 本地也刪除。
同步完成後，可選擇自動重啟播放服務，讓新清單生效。

mpv 建議啟動參數：

請用 mpv IPC 控制播放，例如 pause、playlist-next、playlist-prev、stop、set_property volume 等。

六、安全要求

Web 管理介面必須要登入。
預設帳密第一次登入後要提示修改。
Dropbox token 不可在前端明文顯示。
rclone config 權限需要 chmod 600。
系統預設只建議透過 Tailscale IP 存取，不開公網。
不要做 port forwarding。
所有會執行 shell command 的功能必須白名單化，不允許任意命令注入。
Dropbox 路徑、本地路徑必須做基本驗證。
清空本地音樂、重開機等危險操作需要二次確認。
所有操作寫入 audit log。

七、systemd 服務需求

請建立至少三個服務/排程：

nikko-Music-Hub-web.service
負責啟動 Web 管理介面。
nikko-music-player.service
負責 mpv 自動播放、掛掉自動重啟。
nikko-music-sync.service / nikko-music-sync.timer
負責 Dropbox 定時同步。

播放服務要求：

開機自動播放。
mpv 掛掉自動重啟。
沒有 MP3 時不要無限崩潰刷 log，應顯示「沒有音樂檔」。
同步完成後可重啟播放服務。
支援 mpv IPC 控制。
