## e. 需要手動處理

### e.1 EMQX Dashboard 啟用 HTTPS + 限制 18083

請在 broker 主機（114.55.1.51）執行：

```bash
# 1. 準備 TLS 憑證（可先用 Let's Encrypt 或自簽）
#    假設已經有 /etc/emqx/cert/emqx.pem 與 /etc/emqx/cert/emqx.key

# 2. 編輯 EMQX 設定（/etc/emqx/emqx.conf 或透過 Dashboard）
#    新增 HTTPS listener，並停用 HTTP
dashboard {
  listeners.https {
    bind = "0.0.0.0:18084"
    ssl_options {
      certfile = "/etc/emqx/cert/emqx.pem"
      keyfile  = "/etc/emqx/cert/emqx.key"
      cacertfile = "/etc/emqx/cert/ca.pem"
    }
  }
  # 若要完全關閉 HTTP，註解掉下面這行
  # listeners.http.bind = "0.0.0.0:18083"
}

# 3. 重啟 EMQX
sudo systemctl restart emqx

# 4. 防火牆：只允許你的管理 IP 連 18083，其餘封鎖
sudo iptables -I INPUT -p tcp --dport 18083 -s <你的管理IP> -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 18083 -j DROP

# 5. 測試
# 瀏覽器開 https://114.55.1.51:18084，HTTP 18083 從其他 IP 應被拒絕
```

### e.2 重新簽發 EMQX broker TLS 憑證（含 SAN + AKI）

在持有 Root CA 私鑰的機器上執行：

```bash
# 1. 建立 openssl.cnf，加入 SAN
#    內容範例：
#    [req]
#    distinguished_name = req_distinguished_name
#    req_extensions = v3_req
#    [req_distinguished_name]
#    [v3_req]
#    subjectAltName = @alt_names
#    [alt_names]
#    DNS.1 = Server
#    IP.1 = 114.55.1.51
#    authorityKeyIdentifier = keyid:always,issuer:always

# 2. 產生 CSR
openssl req -new -newkey rsa:2048 -nodes \
  -keyout emqx-server.key -out emqx-server.csr \
  -subj "/C=TW/O=Nikko/CN=Server" -config openssl.cnf

# 3. 用 Root CA 簽署
openssl x509 -req -in emqx-server.csr -CA root-ca.pem -CAkey root-ca.key \
  -CAcreateserial -out emqx-server.crt -days 365 \
  -extensions v3_req -extfile openssl.cnf

# 4. 把 crt + 中繼 CA 組成完整鏈
#    確保 emqx-server.pem 包含 server cert + intermediate CA

# 5. 上傳到 broker，更新 emqx.conf
#    ssl.certfile = "/etc/emqx/cert/emqx-server.pem"
#    ssl.keyfile  = "/etc/emqx/cert/emqx-server.key"
#    ssl.cacertfile = "/etc/emqx/cert/root-ca.pem"

# 6. 重啟 EMQX
sudo systemctl restart emqx
```

> Prompt 給 GPT：「請教我如何為 EMQX 產生包含 SAN=DNS:Server/IP:114.55.1.51 與 AKI 的完整 TLS 憑證鏈，並用 Root CA 私鑰簽署。」
