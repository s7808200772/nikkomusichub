## b. 本次變動項目

| 序號 | 項目 | 改變內容 | 如何驗收 | 預期效果 |
|---|---|---|---|---|
| 1 | 修復 Pi SQLite 連線關閉 | `app/routes/auth.py` 與 `app/routes/health.py` 移除對 `get_db()` 回傳連線的 `conn.close()`，避免關閉執行緒區域快取連線 | 在 Pi 重新安裝或執行 `repair-pi.sh` 後，`systemctl status nikko-music-hub-web.service` 為 active | Pi web service 可正常啟動 |
| 2 | Cloud 側邊欄選項改四字 | `components/Sidebar.js` 導航改為「控制中心 / 店點管理 / 監控紀錄」 | 開啟 Cloud 任一頁面，左側選項皆為四字 | 符合需求且排版一致 |
| 3 | Dashboard 四張卡片一列 | `DashboardClient.js` 改用 `.stats-grid` 四欄排版，移除「前往批量指令」連結 | 開啟 `/`，上方統計卡片在一列 | 視覺更緊湊 |
| 4 | Dashboard 告警/店點列表改條列 | `DashboardClient.js` 最近告警與店點列表改用 `.list-table` 表格 | `/` 頁面下方列表以表格呈現 | 資訊更易掃描 |
| 5 | Commands 單店卡片改條列 | `CommandsClient.js` 改以表格列顯示店點、最後結果、四類指令按鈕、可展開輸出 | 開啟 `/commands`，每家店為一列 | 多店操作更方便 |
| 6 | Alerts 確認按鈕語意與表格化 | `AlertsClient.js` 按鈕改為「標記已處理」，並加說明文字；告警列表改為表格 | `/monitoring` → 告警中心 | 避免誤解按鈕功能 |
| 7 | 遠端 Log 頁面 Pi 風格 | `LogsClient.js` 新增統計卡片、分頁 tabs、搜尋、等級過濾、分頁表格、CSV 匯出 | `/monitoring` → 遠端 Log，選擇店點後載入 | 操作體驗接近 Pi `/logs` |
| 8 | 修復 Tabs 不渲染內容 | `components/Tabs.js` 增加 `children` prop 並在 tab 列下方渲染 | 所有使用 Tabs 的頁面（Stores/Monitoring）內容正常顯示 | 分頁功能恢復 |
| 9 | Cloud Production 重新部署 | 已 push `security-final` 並執行 `vercel --prod` | 開啟 `https://cloud-vercel-xi.vercel.app` 可看到新版 UI | 線上環境同步 |

## e. 預計執行項目

1. 根據 Pi 執行 repair/diagnose 腳本的輸出，確認 `sqlite3.ProgrammingError` 已解決，或進一步修正。
2. Pi web service 正常後，測試 Cloud → Pi MQTT 指令與遠端 Log。
3. 取得 LINEAR_API_TOKEN 後補上 Linear project update。
4. 根據實際使用回饋，微調 Cloud 表格欄寬與 RWD。
